## Анализ матрицы

Это небольшая консольная утилита на TypeScript, которая генерирует квадратную числовую матрицу, анализирует каждую строку и выводит форматированный отчёт в терминал.

Программа:
- **Генерирует** матрицу \(10 \times 10\) из случайных целых чисел в диапазоне \[MIN, MAX\] (по умолчанию \[-100, 100\], см. константу `RANGE` в `main.ts`).
- **Находит** глобальный минимум по всей матрице.
- **Анализирует каждую строку**:
  - Показывает все значения в строке.
  - Вычисляет **минимальное положительное значение** в строке (или `—`, если положительных чисел нет).
  - Считает **минимальное количество замен**, необходимых для того, чтобы в строке не было трёх подряд идущих чисел одного знака (положительных/отрицательных). Логика обработки нулей подробно описана в комментарии внутри функции `calculateFixes` в `main.ts`.
- **Помечает** все строки, содержащие глобальный минимум, символом `*` в консольном выводе.

Результатом работы является аккуратно отформатированная таблица в консоли с колонками: индекс строки, значения строки, минимальное положительное число и количество необходимых замен.

## Требования

- **Node.js** (подойдёт любая актуальная LTS-версия)
- **npm** (идёт в комплекте с Node.js)

## Установка

Из корневой директории проекта выполните команду:

```bash
npm install
```

Это установит зависимости `typescript` и `tsx`, которые используются для прямого запуска файла TypeScript.

## Запуск программы

Из корневой директории проекта:

```bash
npm run start
```

Будет выполнена команда:

```bash
tsx main.ts
```

В терминале появится таблица с результатами анализа матрицы.

## Структура проекта

- `main.ts` — основная логика генерации матрицы, анализа и вывода в консоль.
- `package.json` — конфигурация проекта Node.js и сценарии запуска.

## Настройка

Параметры анализа можно изменить в файле `main.ts`:

- **Размер матрицы**: измените значение `MATRIX_SIZE`, чтобы сделать матрицу больше или меньше.
- **Диапазон значений**: скорректируйте `RANGE.MIN` и `RANGE.MAX`, чтобы изменить интервал случайных чисел.

После любых изменений просто снова выполните `npm run start`, чтобы увидеть обновлённое поведение программы.
